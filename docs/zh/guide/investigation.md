# 调查与证据

ThreatIncident 是 V3il 的调查主线。它将相关环境、行为、检测结果、任务、证据、分析、动态诱导和报告组织在同一上下文中，帮助团队理解一次攻击活动如何发展，以及当前结论是否足以支撑响应。

## Incident 视图

一个 Incident 主要回答以下问题：

- 哪些环境和时间段受到影响；
- 攻击行为之间存在什么关系；
- 当前调查覆盖了哪些问题，还有哪些缺口；
- 证据支持怎样的攻击意图、攻击链和画像；
- 风险与响应优先级如何判断；
- 哪些环境调整已经执行，结果如何；
- 当前结论是否达到报告和关闭要求。

新的重要行为会进入现有调查或形成新的 Incident。调查进入收尾阶段后仍可因新证据恢复。

## 五智能体协作

| 编码 | 名称 | 调查职责 |
| --- | --- | --- |
| `cso` | V3il | 制定调查计划，协调专家，处理冲突，复核结论并管理 Incident 进程。 |
| `cth` | H4wk | 重建行为、时间线、攻击过程、技术路径和意图。 |
| `cde` | Ph4ntom | 评估并实施有助于验证假设的环境调整。 |
| `cie` | L1ly | 整理指标与外部上下文，形成画像并说明归因边界。 |
| `cir` | J4ck | 判断风险、停止条件、响应优先级和防御改进。 |

V3il 负责跨角色协调。专家工作围绕明确任务展开，减少重复分析和缺乏依据的结论。

## 调查计划

调查任务包含四个核心要素：

1. **问题：** 需要回答的安全问题；
2. **范围：** 与问题直接相关的行为和上下文；
3. **负责人：** 最适合处理该问题的智能体角色；
4. **完成标准：** 需要提供的证据、分析或决策。

任务可以存在依赖关系，例如行为重建结果为攻击意图评估提供输入，目标数据确认结果为环境调整决策提供依据。V3il 根据调查进度协调这些依赖，并对专家提交的结果进行复核。

## 协作连续性

每个 Incident 拥有一个权威 Agent Session，由 Incident 工作区、Agent Console 和运营视图共同使用。调查任务说明、操作者决策、专家结论和当前协作状态都会沿用这一个 Session，并在所有入口保持一致。

V3il 维护主调查上下文，并将边界明确的工作分配给专家 Run。专家会收到任务范围、相关证据和预期成果。主 Run 需要专家结果时进入等待，并在指定子 Run 完成后继续。交付关系与确定的父子 Run 绑定，使并行专家结论始终进入对应的调查步骤。

操作者可以中断正在执行的 Run，也可以取消 Session 中的全部活动工作。等待依赖的工作仍然支持取消。已经完成的结论、接受的上下文和持久化事件历史会继续保留，为下一次 Run 提供清晰、可复核的起点。

## 证据标准

证据记录连接原始行为、调查任务和分析陈述。有效证据应当具备：

- 清楚的来源和时间；
- 与调查问题直接相关；
- 能够解释其支持或削弱的判断；
- 与其他证据之间的关系明确；
- 保留必要的原始上下文；
- 可由其他操作者复查。

行为记录包含完整性信息，用于发现缺失、乱序或被替换的数据。证据覆盖和数据完整性会参与任务复核与最终报告检查。

## 统一时间线

Incident 时间线将行为事件、审计事件、调查任务、证据记录和欺骗环境版本组织在同一个有序视图中。每个条目保留来源身份和发生时间，操作者可以从一项决策回到支撑它的证据与当时的环境状态。

时间线在不同类型条目之间使用稳定位置进行翻页。新活动可以持续进入时间线头部，操作者仍可连续查看更早历史。这一机制同时满足实时调查中的连续浏览和事后复盘中的顺序还原。

## 分析成果

调查过程维护五类主要成果：

- **攻击意图：** 攻击者当前目标、所处阶段、可信度和下一步假设；
- **攻击链：** 按时间和因果关系整理的行为路径、证据步骤与已知缺口；
- **威胁指标：** 可用于检索、检测和响应的网络、主机、身份或工具指标；
- **攻击者画像：** 目标偏好、能力、技术习惯、工具、基础设施和归因边界；
- **风险评估：** 影响、紧迫度、停止条件、响应建议、防御改进和残余风险。

每次重要修订都会生成新版本。Incident 工作区展示当前结论，也保留历史判断和变化原因。

## 复核与审计

V3il 复核专家结果时关注任务范围、证据覆盖、结论一致性和未解决缺口。操作者可以查看任务分配、智能体运行、环境调整、分析版本、Incident 状态和报告发布记录。

审计时间线用于回答谁在何时做了什么、依据是什么，以及该动作如何影响后续调查。

## 报告与关闭

进入报告阶段前，平台检查：

- 重要行为已经纳入调查范围；
- 关键任务已经完成复核；
- 攻击意图、攻击链、画像和风险结论相互一致；
- 结论对应的证据可用且完整；
- 正在运行的环境调整和专家工作已经结束；
- 报告引用了确定的分析和证据版本。

最终报告发布后，平台可以生成证据包并将内容沉淀到 LightRAG。报告最终化会固定分析与证据范围，并将知识发布纳入独立恢复链路；已经完成的报告持续向操作者开放。Incident 关闭后仍保留时间线、证据、分析、报告和审计记录，供后续复盘与检索。
